 REQUEST FLOW (numbered)
═══════════════════════════════════════════════════════════════════════════════

 1.  Client sends  HTTP + Bearer <JWT>  to Gateway :8080
 2.  Gateway validates JWT signature using locally cached Keycloak JWKS
 3.  Rate-limit bucket checked in Redis (per client IP)
 4.  X-Correlation-ID injected (generated or forwarded)
 5.  Route resolved via Eureka lb:// to healthy service instance
 6.  Microservice re-validates JWT claims/roles (defense-in-depth)
 7.  Business logic executes → Kafka event published if state changed
 8.  Downstream consumers (Notification, Recommendation) react asynchronously
 9.  Response returns through Gateway with X-Correlation-ID in headers

═══════════════════════════════════════════════════════════════════════════════
 SECURITY LAYERS
═══════════════════════════════════════════════════════════════════════════════

 Layer 1 — Network:   TLS everywhere (HTTPS + internal mTLS in production)
 Layer 2 — AuthN:     Keycloak JWT (RS256, short-lived 5 min access tokens)
 Layer 3 — AuthZ:     Gateway blocks unauthenticated requests
 Layer 4 — Service:   Each service re-checks JWT roles independently
 Layer 5 — Rate:      Redis token-bucket limits per IP (50 req/s burst 100)
 Layer 6 — CORS:      Gateway enforces allowed origins list
